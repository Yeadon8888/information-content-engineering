# 内容系统参考

## 素材库结构与检索

`content-system/03-素材库/` 按以下类别组织：

```
03-素材库/
├── _索引.md          # 快速检索入口
├── 开头钩子.md
├── 结构框架.md
├── 口播脚本框架.md
├── 标题钩子.md
├── 认知表达.md
├── 金句表达.md
├── 案例素材.md
├── AI工具评测模板.md
└── [独立素材包].md   # 由文章链接 / 图片链接 / 推文链接沉淀出的专题素材包
```

**检索规则**：先读 `_索引.md` 判断哪些文件相关，只读相关文件，不全量加载。找到后展示给用户并建议复用；找不到直接告知，不强行匹配。

---

## 链接入库规则

当用户发来以下内容时，默认不要只在对话里消费掉：

- 文章链接
- 图片链接
- 推文 / X 链接
- 可公开访问的案例页、产品页、活动页

按下面规则处理：

### 1. 先分流

- **只是线索，还没形成稳定观点** → 记进 `content-system/01-选题库/_待处理.md`
- **已经能提炼成后续反复调用的内容资产** → 写成独立 `素材包`，放进 `content-system/03-素材库/`

### 2. 素材包命名

文件名建议：

`[主题关键词]-素材包.md`

例如：

- `Naval-人人构建自己的应用-素材包.md`
- `OpenAI-Agent-工作流化趋势-素材包.md`
- `某产品截图拆解-素材包.md`

### 3. 素材包最小结构

```markdown
# [标题]

> 来源：[来源说明]
> 链接：[原始链接]
> 日期：YYYY-MM-DD
> 类型：[趋势观点 / 产品案例 / 配图参考 / 选题线索]

## 一句话主题

## 原始信息整理

## 可复用判断 / 观点 / 案例

## 可直接用的标题方向

## 适用场景

## 原始链接归档
```

### 4. 索引同步

凡是写进 `03-素材库/` 的独立素材包，都要同步往 `_索引.md` 追加一行，方便后续检索。

原则：**先保住链接，再提炼结构；能直接成资产的，不要只塞进待处理。**

## 共享资产 vs 专属资产

### 共享资产

这些资产可以被文章链路和口播链路共同复用：

- 开头钩子
- 标题钩子
- 认知表达
- 金句表达
- 案例素材
- AI 工具评测模板

### 文章链路专属资产

- 正文结构展开
- 深度分析节奏
- 图文阅读型收尾

优先参考：`references/写作方法.md`、`references/深度写作.md`、`03-素材库/结构框架.md`

### 口播链路专属资产

- 口播推进节奏
- 分镜模板
- 画面/字幕/音效设计

优先参考：`references/口播脚本.md`、`03-素材库/口播脚本框架.md`

原则：素材与观点共享，最终结构和表达链路分开。

**沉淀归属**：

| 素材类型 | 写入文件 |
|---|---|
| 开头写法 | `开头钩子.md` |
| 文章整体结构、段落节奏 | `结构框架.md` |
| 分镜模板、口播节奏、文章改口播方法 | `口播脚本框架.md` |
| 可传播的句式、比喻 | `金句表达.md` |
| 真实案例、数据 | `案例素材.md` |

**素材写入格式**：

```markdown
## [素材标题]

> 来源：[文章标题] | 数据：[相关表现] | 日期：YYYY-MM-DD

[素材内容]

**适用场景**：[什么时候用]
```

每次新增素材后，在 `_索引.md` 追加一行：`- [类别] [素材标题] → [文件名.md]`

## 一条主内容，多次分发

一篇内容最好被当成“母稿”，而不是“一次性成品”。默认可拆成：

- 公众号文章
- 纯口播稿
- 分镜脚本
- 标题/封面文案
- 金句卡片
- 拆条短内容

沉淀时优先回收这些中间件：

- 开头钩子
- 核心结构
- 金句
- 好案例
- 成绩最好的 CTA

如果一次创作同时产出了文章和口播，优先把共用骨架沉淀成素材，而不是把两份完整成稿都硬塞进素材库。


## 方法论沉淀格式

在 `content-system/04-方法论/` 对应文件追加：

```markdown
## [规律名称]

**触发条件**：
**规律描述**：
**数据支撑**：[来自哪篇文章，表现如何]
**实操方法**：

> 更新于：YYYY-MM-DD
```

**数据判断标准**：

| 表现 | 行动 |
|---|---|
| 显著高于平均（点赞/在看 ×2 以上） | 立即沉淀到素材库和方法论 |
| 与平均持平 | 记录数据，观察下次复用效果 |
| 显著低于平均 | 分析原因，写入 `避坑记录.md` |

`04-方法论/` 推荐文件：`_迭代记录.md`、`标题规律.md`、`开头规律.md`、`结构规律.md`、`避坑记录.md`

## 初始化内容系统

执行 `/初始化内容系统` 时创建：

1. `content-system/01-选题库/_待处理.md`
2. `content-system/03-素材库/_索引.md`（含分区：开头钩子/结构框架/口播脚本/标题钩子/认知表达/金句表达/案例）
3. `content-system/04-方法论/_迭代记录.md`

原则：系统是用出来的，不是设计出来的。从第一篇文章开始积累，边用边迭代。
