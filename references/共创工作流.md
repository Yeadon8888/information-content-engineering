# 渐进式共创工作流

适用于“我们一起把这条内容做出来”，而不是“你直接代写完给我”。

默认模式下，每一轮只推进一个关键决策，避免一次给作者太多东西。用户没有明确指定模式时，优先走这里。

## 三种协作模式

### 1. 渐进共创模式（默认）

适合新选题、新系列、新产品、新观点。

顺序：

1. 先定主题判断和目标受众
2. 再定内容承诺和平台
3. 再给 2-3 个开头/角度
4. 再给结构骨架
5. 再写一段样稿
6. 最后扩成完整稿

每一轮最好只让作者做一个决定，比如“选哪个角度”“要不要更锋利一点”“这一段偏故事还是偏干货”。不要把标题、结构、全文、分发版本一次全抛出来。

### 2. 快速直出模式

只有在用户明确表示“直接给我整篇”“先出一版初稿”时使用。

即便直出，也要在内部先完成下面四件事：

- 判断目标平台
- 提炼一句话承诺
- 选一个结构骨架
- 明确收尾动作

缺这四步的直出，通常只是长文本堆砌。

### 3. 编辑精修模式

适合用户已经有草稿、提纲、录音转写稿、文章链接、口播样本。

优先顺序：

1. 先诊断问题属于钩子、结构、语气还是节奏
2. 先改最影响传播的那一层
3. 再决定要不要重写全文

不要一上来整稿推翻。先判断原稿里能保留什么。

## 共创推进顺序

### 第一层：内容承诺

先产出这四项中的大部分：

- 这条内容讲给谁
- 它解决什么问题
- 观众为什么现在就该看
- 看完后会得到什么

如果这层不清楚，后面的标题和结构都会漂。

### 第二层：切入角度

通常给 2-3 个方向就够了，避免选择疲劳。每个方向至少说清：

- 这条线的开头怎么抓人
- 中段靠什么推进
- 结尾能落到什么价值

### 第三层：结构骨架

不要急着写满。先把内容拆成几个推进单元，例如：

- `钩子 -> 痛点 -> 解法 -> 结果 -> CTA`
- `判断 -> 解释 -> 案例 -> 升维 -> 收尾`
- `开场脑洞 -> 演示 -> 对普通人的意义`

作者对结构点头之后，再写正文。

### 第四层：局部样稿

优先写最关键的一段：

- 文章：开头 150-300 字
- 口播：前 20-40 秒
- 分镜：前 3-5 个镜头

这是最低成本验证方向的方法。

### 第五层：完整成稿

完整稿不是终点，还要继续补：

- 标题/封面文案
- 首评/摘要/导语
- 金句
- CTA
- 多平台改写版本

## 针对文章的共创节奏

推荐顺序：

1. 选题判断
2. 标题方向
3. 结构大纲
4. 开头样稿
5. 正文扩写
6. 结尾与互动
7. 去AI化

如果用户说“先别写全文”，就停在当前层，不越界。

## 针对口播的共创节奏

推荐顺序：

1. 一句话卖点
2. 前 3 秒开头
3. 中段推进方式
4. 是否需要 B-roll/录屏/截图
5. 收尾 CTA
6. 口语化压缩

口播最先验证的不是信息完整，而是“第一句能不能把人拉住”。

## 何时主动做假设

当用户没有给足信息时，可以默认假设：

- 平台没说：优先按公众号文章或短视频通用稿处理
- 时长没说：口播默认 60-90 秒
- 风格没说：沿用作者配置里的伊登风格
- 是否要分镜没说：先给纯口播稿，再补一句可继续扩成分镜版

能继续就继续，不要因为信息不全卡住协作。
